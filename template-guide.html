עקרון המאסטר — שינוי אחד, עדכון לכולם

הטמפלט cs-v1.html הוא המאסטר. כל עמוד פרויקט הוא העתק של המאסטר, ולא קישור חי אליו. לכן שינוי עיצובי כולל דורש עדכון ידני בשני שלבים:

1

עדכן את cs-v1.html

בצע את השינוי העיצובי (צבע, גופן, spacing, רכיב חדש) בקובץ המאסטר.

2

הפץ לכל הפרויקטים

העתק את בלוק ה-CSS/JS שעדכנת מ-cs-v1.html לכל קבצי הפרויקטים (agents.html, dspm.html וכו').

טיפ: שמור את כל ה-CSS וה-JS בראש הקובץ בבלוקים מסומנים — כך קל לזהות מה להעתיק.

כל חלק באתר יש לו קובץ טקסט משלו — עורכת שם, מעלה לקלוד, מקבלת עמוד מעודכן:

portfolio/ ├── main-content.txt ← כותרות קארדים + סדר + ניווט ├── about-content.txt ← כל תוכן About (dark+light) ├── agents-content.txt ← כל תוכן פרויקט 1 ├── dspm-content.txt ← כל תוכן פרויקט 2 ├── violation-pane-content.txt← כל תוכן פרויקט 3 ├── [project]-content.txt ← פרויקטים 4-6 (ייצרו בבנייה) └── cs-v1.html ← המאסטר (עיצוב בלבד)

עמוד About — מצב כהה ובהיר

הקובץ about.html הוא קובץ אחד שתומך בשני מצבים. הוא מחליף מצב לפי פרמטר ב-URL:

קישורמצבמאיפה?
about.htmlDark (ברירת מחדל)index.html — כפתור About בסיידבר
about.html?mode=lightLightעמודי הפרויקטים — כפתור About בסיידבר
שינוי תוכן: כדי לעדכן את עמוד About (טקסט, ניסיון, קישורים) — ערכי רק את about.html. שני המצבים יתעדכנו אוטומטית כי זה אותו קובץ.

⭐ סדר הפרויקטים — קובץ שליטה אחד

כל סדר הפרויקטים נשלט מקובץ אחד בלבד: projects-order.js. אין צורך לגעת בשום קובץ אחר.

הקובץ הזה שולט על שלושה דברים בו-זמנית:
1. סדר הקארדים בעמוד הבית (שמאל-לימין, שורה אחר שורה)
2. סדר כפתורי ← Previous / Next → בתוך כל עמוד פרויקט
3. איזה פרויקט הוא "קארד 1" — הראשון, ולכן ללא כפתור "Previous work"

החוק של קארד 1

הפרויקט שראשון ברשימה ב-projects-order.js הוא תמיד "קארד 1" — תחילת המסע. הוא לא מציג "← Previous work", רק "Next work →". זה אוטומטי לחלוטין — אם תזיזי פרויקט אחר לראש הרשימה, הוא יקבל את ההתנהגות הזו, והפרויקט הקודם יקבל בחזרה את כפתור ה-Previous.

איך משנים סדר

פותחים את projects-order.js, ומסדרים מחדש את השורות. זהו.

window.PROJECTS = [
  'agents.html',             // קארד 1 — ללא Previous
  'dspm.html',               // קארד 2
  'violation-pane.html',     // קארד 3
  'retraining-model.html',   // קארד 4
  'security-dashboard.html', // קארד 5
  'logos-art-for-fun.html'   // קארד 6 (שורה אחרונה — ללא פסיק)
];

לדוגמה — כדי ש-DSPM יהיה קארד 1: מעבירים את 'dspm.html' לראש הרשימה. כל השאר זזים אוטומטית, DSPM מאבד את כפתור ה-Previous, ו-agents (עכשיו שני) מקבל אותו.

חשוב: כל שורה היא שם הקובץ במרכאות + פסיק. השורה האחרונה ללא פסיק. שמרי על הסוגריים המרובעים [ ].

הוספת פרויקט חדש (למשל בעוד שנתיים)

1

צרי את העמוד

שכפלי את cs-v1.html, שני שם (למשל new-project.html), ומלאי תוכן.

2

הוסיפי קארד ב-index.html

העתיקי בלוק <a class="card"> קיים, הדביקי בתוך הגריד, ועדכני: href, data-name="new-project" (חייב להתאים לשם הקובץ), והטקסטים.

3

הוסיפי תמונת קארד ב-index.html

ב-<style> הוסיפי שורה אחת:
.card[data-name="new-project"] .card-img { background-image: url('portfolio images/Main-images/YOUR-IMAGE.png'); }

4

הוסיפי לרשימה

ב-projects-order.js הוסיפי 'new-project.html' במיקום הרצוי. רוצה שיהיה קארד 1? שימי בראש — כל השאר יזוזו ימינה, והפרויקט החדש יהפוך לתחילת המסע אוטומטית.

שינוי טקסט / תמונה של קארד

ב-index.html, מצאי את הקארד לפי data-name (למשל data-name="dspm"):

<a href="dspm.html" class="card" data-name="dspm">
  <div class="card-img"></div>
  <div class="card-overlay">
    <span class="card-title">Cloud data security</span>  <!-- כותרת -->
    <span class="card-sub">Laminar</span>              <!-- תת-כותרת -->
  </div>
</a>

לשינוי תמונה — מצאי ב-<style> את השורה עם אותו data-name והחליפי את שם קובץ התמונה. התמונות קשורות לפי שם הפרויקט (לא לפי מיקום), כך ששינוי סדר לעולם לא שובר את מיפוי התמונות.

למה זה כל כך פשוט: התמונות וה-Previous/Next קשורות לשם הפרויקט (data-name), לא למיקומו. לכן הזזת קארד אף פעם לא שוברת כלום — רק הסדר משתנה.
