## חלק א' — להכניס הכול לריפו (פעם אחת)

### שלב A1 — אספי את הקבצים במקום אחד
על המחשב, פתחי את תיקיית הפרויקט שלך (זאת עם `index.html`, `agents.html` וכו'
והתיקיות `portfolio images`, `case-study-presentation`). הוסיפי לתוכה את הקבצים
החדשים שיצרתי, **בדיוק במבנה הזה**:

```
(שורש הפרויקט)
├── index.html, about.html, agents.html ... logos-art-for-fun.html
├── cs-v1.html
├── portfolio images/ , case-study-presentation/
├── START-HERE.md          ← חדש
├── CLAUDE.md              ← חדש
├── docs/                  ← חדש (4 קבצים)
│   ├── DESIGN-SYSTEM.md
│   ├── RESPONSIVE.md
│   ├── GALLERY-SAPIR.md
│   └── PRESENTATION.md
└── .claude/skills/        ← חדש (2 סקילים)
    ├── new-case-study/SKILL.md
    └── site-ops/SKILL.md
```

> **למה המבנה הזה:** קלוד קורא את `CLAUDE.md` מהשורש ואת הסקילים מ-`.claude/skills/`
> אוטומטית. אם הם לא במקום הזה — הוא לא "יראה" אותם.

### שלב A2 — העלי הכול ל-GitHub
הריפו שלך: **`sapir7106/portfolio`** (כבר קיים).

הדרך הכי קלה לטווח ארוך — **GitHub Desktop** (תוכנה חינמית עם כפתורים, בלי טרמינל):
1. הורידי מ-[desktop.github.com](https://desktop.github.com) והתחברי עם החשבון שלך.
2. `File → Clone repository` → בחרי `sapir7106/portfolio` → בחרי תיקייה במחשב.
3. העתיקי את כל קבצי הפרויקט לתוך התיקייה שנוצרה.
4. ב-GitHub Desktop יופיעו כל השינויים. למטה כתבי תיאור קצר (למשל
   `add memory files + docs`) ולחצי **Commit to main**, ואז **Push origin**.

> **למה Git/commit/push:** *commit* = "צילום מצב" שמור של הפרויקט (נקודת שמירה
> שתמיד אפשר לחזור אליה). *push* = שולח את הצילום הזה ל-GitHub בענן. מהרגע הזה
> העבודה שלך מגובה ולא תלויה במחשב אחד.

(אם את מעדיפה בלי תוכנה: ב-github.com/sapir7106/portfolio → `Add file → Upload
files` → גררי הכול → `Commit changes`. רק שימי לב — בגרירה לדפדפן צריך לפעמים
להעלות תיקייה-תיקייה.)

## חלק ב' — איך עובדים נכון (כל יום עבודה)

### שלב B1 — פותחים סשן
היכנסי ל-**claude.ai/code**, ובחרי את הריפו `portfolio`.
> **למה זה קסם:** קלוד קורא מיד את `CLAUDE.md` ואת הסקילים — אז הוא כבר "מכיר" את
> כל מערכת העיצוב, המבנה, והכללים שלך. את לא צריכה להסביר מאפס.

### שלב B2 — נותנים משימה אחת ממוקדת
לדוגמה: *"In `logos-art-for-fun.html`, replace the GALLERY-01 placeholder with a
real image at this path. Don't change anything else."*
> **למה אחת בלבד + "don't change anything else":** זה מה שמונע את באג ה"נמחק לי X".
> משימה קטנה = שינוי קטן = קל לבדוק.

### שלב B3 — בודקים את ה-diff
קלוד עובד על branch ומראה **diff** (ירוק = נוסף, אדום = הוסר). קראי אותו.
> **למה:** ה-diff הוא הביטוח שלך — את רואה **בדיוק** מה השתנה לפני שזה נכנס.

### שלב B4 — ממזגים ומפרסמים
צרי Pull Request → ב-GitHub לחצי **Merge** ל-`main` → Cloudflare מפרסם תוך דקה.
> **למה PR + merge:** כל מיזוג הוא נקודת שמירה נקייה. אם משהו השתבש — פותחים את ה-PR
> ורואים/מבטלים בדיוק את מה שהשתנה.

### שלב B5 — שתי הרגלים ששווים זהב
- בסוף משימה: *"update the Current status in CLAUDE.md and commit."*
- כשלמדת/החלטת משהו: פשוט אמרי **"תתעד את זה"** — קלוד יבחר את הקובץ הנכון לפי
  המפה ויעדכן בעצמו. (זה ה"תיעוד-עצמי".)
